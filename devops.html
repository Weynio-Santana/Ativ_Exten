Cultura e Ferramentas DevOps

Automatize e integre os processos de desenvolvimento e operações.

O que é DevOps?

DevOps é a combinação de filosofias culturais, práticas e ferramentas que aumenta a capacidade de uma organização de entregar aplicações e serviços em alta velocidade. O objetivo é unir as equipes de Desenvolvimento (Dev) e Operações (Ops) para automatizar e integrar seus processos. Isso resulta em um ciclo de desenvolvimento mais rápido, maior qualidade de software e a capacidade de responder melhor às necessidades do negócio. Um conceito central em DevOps é o pipeline de CI/CD (Integração Contínua/Entrega Contínua).

Docker

Sobre Docker: Docker é uma plataforma de código aberto que automatiza a implantação de aplicações dentro de contêineres. Um contêiner empacota o código de uma aplicação e todas as suas dependências, garantindo que ela funcione de forma rápida e confiável em qualquer ambiente de computação. Diferente das máquinas virtuais, os contêineres são leves e compartilham o kernel do sistema operacional, tornando-os muito mais eficientes.

Ideias de Projetos com Docker:

  • Criar um Dockerfile para "containerizar" uma aplicação web simples (ex: Python/Flask ou Node.js/Express).
  • Orquestrar múltiplos contêineres (ex: uma aplicação e um banco de dados) usando Docker Compose.
  • Publicar uma imagem Docker customizada no Docker Hub.
  • Configurar um ambiente de desenvolvimento local consistente usando contêineres.

Kubernetes (K8s)

Sobre Kubernetes: Também conhecido como K8s, o Kubernetes é uma plataforma de código aberto para orquestração de contêineres. Quando você tem muitas aplicações em contêineres, o Kubernetes ajuda a gerenciá-las em escala: ele automatiza a implantação, o escalonamento e a operação desses contêineres. Originalmente desenvolvido pelo Google, o Kubernetes é o padrão de mercado para gerenciar aplicações complexas e distribuídas.

Ideias de Projetos com Kubernetes:

  • Configurar um cluster Kubernetes local usando Minikube ou Kind.
  • Fazer o deploy de uma aplicação em contêiner no seu cluster local.
  • Expor uma aplicação para acesso externo usando um Service do Kubernetes.
  • Estudar e implementar diferentes estratégias de deploy (ex: Rolling Update).

Git

Sobre Git: Git é um sistema de controle de versão distribuído, gratuito e de código aberto. É uma ferramenta fundamental para qualquer desenvolvedor, permitindo rastrear as alterações no código-fonte ao longo do tempo. Com o Git, várias pessoas podem trabalhar no mesmo projeto simultaneamente sem sobrescrever o trabalho umas das outras. Conceitos essenciais incluem repositórios, commits, branches e merges. Plataformas como GitHub, GitLab e Bitbucket são baseadas no Git.

Ideias de Projetos com Git:

  • Iniciar um repositório para um projeto pessoal e fazer seus primeiros commits.
  • Criar e trabalhar com branches para desenvolver uma nova funcionalidade.
  • Colaborar em um projeto de código aberto no GitHub, fazendo um fork e um pull request.
  • Aprender a resolver conflitos de merge.
